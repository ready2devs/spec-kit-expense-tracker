# prueba_spec Constitution

## Core Principles

### I. Dark Mode by Default
The user interface must be designed with a dark theme as the default preference. High contrast, accessible colors, and modern aesthetics (vibrant colors, glassmorphism, dynamic animations) should be prioritized to create a premium impression.

### II. Clean and Modular Code
Code must be organized into small, reusable, and self-contained components or modules. Follow SOLID principles, maintain high readability, and avoid large, monolithic files. Logic should be separated from presentation where possible.

### III. Modern Next.js Standards
Strictly adhere to the latest stable Next.js features and best practices. This includes using the App Router, Server Components for performance, Client Components only when interactive state is required, and following optimized data fetching and caching patterns.

## Development Standards

All implementation tasks must prioritize visual excellence and performance. Placeholder content is prohibited; high-quality assets (images/icons) should be generated or sourced as part of the task.

## Project Structure

The project follows a standard Next.js directory structure. All feature-specific logic should be encapsulated within the `app/` directory or dedicated `components/` and `lib/` folders.

## Governance
This Constitution supersedes all other project practices. Any amendments require explicit documentation and a version increment. Compliance is mandatory for all feature implementations and code reviews.
